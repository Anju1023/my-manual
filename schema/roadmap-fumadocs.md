# 🍞 ベーカリーマニュアル リニューアル計画 (Fumadocs 版)

独自のコンポーネントや CSS は廃止！Fumadocs の標準機能をフル活用して、シンプルかつメンテナンスしやすい構成に作り直します。

## 🛤 全体スケジュール

### 🟢 Phase 1: 新居の準備 (Setup)

まずは Fumadocs のクリーンな環境を用意します。

- [ ] **プロジェクト作成:** `npm create fumadocs-app` で最新環境を構築。
- [ ] **Tailwind CSS v4:** Fumadocs は標準で対応しているので、そのまま利用。
- [ ] **不要なファイルの排除:** 前回の `My~` コンポーネントや `globals.css` は持ち込まない！

### 🟡 Phase 2: コンテンツの再構築 (Re-writing)

ここがメイン作業！旧マニュアルを「原稿」として見ながら、1 ページずつ丁寧に作り直します。

- [ ] **フォルダ構成の再設計:** `content/docs/` 以下に、分かりやすい階層を作る。
- [ ] **MDX の書き換え:**
  - `MyCallout` → Fumadocs 標準の `<Callout>` に変更。
  - `MySteps` → Fumadocs 標準の `<Steps>` に変更。
  - `MyTable` → 普通の Markdown テーブルまたは標準機能に変更。
- [ ] **メタデータ設定:** `meta.json` を使って、サイドバーの並び順やタイトルを整理。

### 🟠 Phase 3: "あんじゅちゃん色" を少しだけ (Branding)

スタイルはデフォルトでいいけど、アクセントカラーくらいは設定してもいいかも？（後回しで OK！）

- [ ] **テーマカラー設定:** 必要であれば、パン屋さんっぽい色（オレンジやブラウン系）を基本カラーに設定。
- [ ] **ロゴ設定:** ヘッダーに「🍞 ベーカリー業務マニュアル」を表示。

### 🟣 Phase 4: 公開準備 (Deploy)

- [ ] **全文検索の確認:** Fumadocs 標準の検索がちゃんと動くかチェック。
- [ ] **静的ビルド設定:** `output: 'export'` で書き出し。
- [ ] **完成‼️**
